Motivations

So now that we have covered the traditional tools and I previously mentioned that I actively use JavaScript when solving real-world problems, I want to spend some time on the motivations for why I choose JavaScript over other platforms...

Why JavaScript?

One of the first questions, if not THE first question, I am asked when I tell people that I do much of my data analysis work in JavaScript is: Why JavaScript? Why not X? Why is this environment/platform any more suitable than already established platforms built on Python, R, Julia, etc? Any inherent advantages?

Ubiquity

 

JavaScript is everywhere. Almost everything either already runs or supports running a browser. Distribution is streamlined.

  • Githut
  • Module Counts
  • IoT
  • Samsung and iotjs

Performance

The JavaScript runtime environment is an environment which is under constant pressure to run faster. In other languages, you simply choose a different language if you want to go faster; just kind of accepted. With JavaScript, ain't going to go away anytime soon and don't have the luxury to use some other "faster" language. We see all sorts of optimizations from the compiler on up: code invariant loop motion, dead code elimination, cache inlining, etc. These may happen elsewhere, but I hypothesize with much less fervor.

JSON


  {
    "type": "Matrix",
    "dtype": "int8",
    "shape": [5,2],
    "offset": 0,
    "strides": [2,1],
    "raw": true,
    "data": [4,2,13,1,1,8,21,9,9,11]
  }
JSON is a first-class citizen. Often in other languages, considerable time is spent marshalling and unmarshalling data from JSON. While JSON is still marshalled/unmarshalled, support for JSON is built-in and 1:1 correspondence between file structure and program structure. Additionally, JSON facilitates different data access patterns. And lastly, JSON is a predominant transport format for the web and almost any API.

Streams


  source.pipe( transform ).pipe( transform ).pipe( destination );
Streams API: while nothing inherent to JavaScript itself, its prevalence as a programming pattern facilitates a different way of thinking about problems compared to the procedural paradigms found in MATLAB, etc.

CLI


$ cat ./data.csv | node ./bin/filter | node ./bin/stats > ./out.txt
Creating (general purpose) command-line utilities is easy and an accepted practice. Not the case with other environments; e.g., MATLAB, Python, R.
